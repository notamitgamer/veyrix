Icon Guide & Usage

  • Open File Load a text or code file from your device into the IDE.
  • Save Manually save the current file to your browser's local storage.
  • Save Snapshot Create a versioned backup of your current file (maximum 5 snapshots per file).
  • View History View all your saved snapshots and safely restore older versions.
  • Share Share your code directly via device, fast local link, or cloud upload.

Frequently Asked Questions

All your files are stored locally on your device inside your browser's secure IndexedDB. They are never uploaded to any server unless you specifically use the Cloud Share tool.
Fast Share compresses your code using Lempel-Ziv and puts it entirely in the URL without using any server. Cloud Share uploads your code to an anonymous Firebase Firestore database, giving you a shorter, cleaner link.
Yes. Because files are local, no one can see them. If you use Cloud Share, you can add a password. Passwords are securely hashed using the Web Crypto API (SHA-256) locally before being checked against the database.
You can save a maximum of 5 snapshots per file. If you try to save a 6th snapshot, you will be prompted to replace the oldest one.
