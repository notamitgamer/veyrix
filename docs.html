Introduction

Veyrix IDE Documentation

Veyrix IDE is a progressive web application (PWA) designed to provide a fast, offline-capable, and secure coding environment directly in your browser. It emphasizes local data ownership, offering developers a scratchpad that feels native without the overhead of heavy desktop applications or server-side setups.

Blazing Fast

No backend compiling or server latency. Everything executes right in your local DOM environment.

100% Private

Code never leaves your device unless you explicitly invoke the sharing mechanisms.

Editor Capabilities

Powered by Ace Editor, Veyrix provides a desktop-grade writing experience tailored for modern web browsers and mobile devices.

Smart Language Detection

Veyrix automatically detects the syntax based on the file extension or special file names. You do not strictly need extensions to format known files.

  • HTML (.html)
  • CSS (.css)
  • JS/TS (.js, .ts, .jsx)
  • JSON (.json)
  • Markdown (.md)
  • Python (.py)
  • YAML (.yml)
  • Shell (.sh)
  • Dockerfile

Built-in Version Control (Snapshots)

Never lose an iteration. Click the snapshot icon (or use the mobile header) to save a timestamped version of your file. Veyrix holds up to 5 snapshots per file. If you attempt to save a 6th, you will be prompted to safely overwrite the oldest version. Restoring a snapshot will immediately replace your current editor content.

Data Handling & Storage

IndexedDB Architecture

All your source code is securely isolated inside your browser's native IndexedDB instance (Database: VeyrixFS). IndexedDB allows for structured, large-capacity storage completely shielded from external networks.

To clear your data: You can either clear your browser site data manually, or open the Settings modal and click the red "Factory Reset" button to instantly drop the entire database.

Offline PWA Capabilities

Veyrix registers a Service Worker upon your first visit. This caches the HTML, CSS, Ace Editor libraries, and SVG assets. Subsequent visits to Veyrix will load instantly, regardless of your internet connection status. Note: Firebase Cloud Sharing requires network connectivity.

Sharing Mechanics

Distribute snippets seamlessly using one of three integrated sharing pipelines.

Fast Local Share

Utilizes LZString.compressToEncodedURIComponent(). Your raw code string is compacted and shoved directly into the URL parameters. No backend is ever touched.

Secure Cloud Share

Pushes data to a Firebase Firestore collection securely. Optional passwords are hashed via crypto.subtle.digest client-side before transmission.

Native Device Share

On supported mobile and desktop browsers, Veyrix hooks into the navigator.share() API, allowing you to instantly beam code snippets through iMessage, WhatsApp, AirDrop, or your OS clipboard manager.

Architecture & Tech Stack

Veyrix avoids heavy build steps (Webpack/Vite) by relying on a strict single-file architecture supported by powerful CDNs.

  • 🎨
    Tailwind CSS (CDN) Configured dynamically via inline script. Responsible for the responsive drawer layout, dark mode colors, and CSS grid structures.
  • 📝
    Ace Editor (v1.32.6) Loaded from cdnjs. Provides the core tokenization, theme (Tomorrow Night Eighties base), and code beautification extensions.
  • 🔥
    Firebase SDK (v11.6.1) Dynamically imported via ES Modules only if the user clicks the Cloud Share tab, keeping initial load times lightning fast.

Privacy & Security Policies

Are you tracking me?
No. Veyrix utilizes zero analytical tracking. There is no Google Analytics, no PostHog, and no internal metrics logging. The console is clean.

Can the developer see my code?
If you keep your files local, they remain exclusively on your hard drive. If you push code via Cloud Share, it enters a public-facing Firebase Firestore database. The developer (Amit) has administrative access to this database, but the records are entirely anonymous and disconnected from your IP or identity. Do not upload API keys or sensitive secrets using Cloud Share.

Contributing

We welcome community contributions! To contribute to Veyrix IDE, follow these steps:

1. Fork the repository at github.com/notamitgamer/veyrix

2. Clone locally:
$ git clone https://github.com/YOUR_NAME/veyrix.git

3. Create a feature branch:
$ git checkout -b feature/awesome-addition

4. Make changes to the core single-file architecture.

5. Commit and push:
$ git push origin feature/awesome-addition

6. Open a PR detailing your architectural choices.
